Prompting Techniques

Master the art of crafting effective prompts for AI coding assistants

Role-Based Prompting

Assign a specific role to the AI, such as "experienced senior developer" or "UI/UX specialist," to shape the type and quality of code it generates.

Example:

Act as a senior React developer with 10 years of experience specializing in performance optimization. Review the following component and suggest improvements to enhance rendering efficiency and code organization:
Learn More

Chain of Thought

Guide the AI through a step-by-step reasoning process to solve complex coding problems, resulting in better thought-out and more robust solutions.

Example:

Let's create a secure user authentication system in Node.js. 
First, think about what database we should use. 
Then, consider password hashing strategies. 
Next, design the API endpoints needed. 
Finally, implement JWT token generation and validation.
Learn More

Constraint-Based Prompting

Set specific constraints and requirements to narrow the AI's focus and ensure the generated code meets particular standards or patterns.

Example:

Create a Python function that sorts a list of dictionaries based on a specified key with these constraints:
- Must have O(n log n) time complexity
- Must handle missing keys gracefully
- Must be fully type-hinted
- Must include comprehensive docstrings
- Must include unit tests
Learn More

Contextual Prompting

Provide rich context about the project, existing codebase, and architectural decisions to help the AI generate code that integrates seamlessly with your system.

Example:

Our project is a React e-commerce site using Redux for state management, Styled Components for styling, and Jest for testing. We follow Atomic Design principles. Our components are structured as:

[paste folder structure]

Please create a ProductCard component that fits our architecture and styling patterns to display product information consistently.
Learn More

Debugging AI Code

Effective strategies for fixing and improving AI-generated code

Error Analysis Prompts

When AI-generated code contains errors, structure your follow-up prompts to help the AI analyze and fix specific issues while explaining its reasoning.

Example Prompt:

The code you provided is throwing the following error:

[paste error message]

Please:
1. Explain what's causing this error
2. Provide a fix with comments explaining the changes
3. Suggest how I could have prevented this issue

Code Review Requests

Ask the AI to perform a thorough code review on its own generated code, focusing on best practices, edge cases, and potential performance issues.

Example Prompt:

Please review the following code you generated and identify:
- Potential bugs or edge cases not handled
- Security vulnerabilities
- Performance bottlenecks
- Areas that could be refactored for readability
- Missing error handling

For each issue found, explain why it's a problem and suggest a fix.

Implementation Alternatives

When AI-generated code works but doesn't seem optimal, ask for alternative implementations to compare approaches and understand tradeoffs.

Example Prompt:

The solution you provided works, but I'm concerned about [specific issue].
Could you provide 2-3 alternative implementations for this function,
explaining the tradeoffs between them in terms of:
- Performance characteristics
- Memory usage
- Readability and maintainability
- Edge case handling

Progressive Enhancement

Start with a basic working implementation and progressively enhance it through targeted follow-up prompts focusing on specific aspects.

Example Prompt Sequence:

1. First prompt: "Create a basic form validation function for email inputs"
2. Follow-up: "Now enhance it with more comprehensive validation rules"
3. Follow-up: "Add detailed error messages for each validation failure"
4. Follow-up: "Optimize the function for performance and readability"
5. Follow-up: "Add TypeScript typing and JSDoc documentation"

Submission Guidelines

  • Ensure your prompt is clear, specific, and well-structured
  • Include placeholders in brackets [like this] for variables
  • Provide a descriptive title that clearly indicates what the prompt creates
  • Test your prompt with multiple AI tools before submitting
  • Avoid submitting prompts that may generate harmful, unethical, or insecure code
  • Provide context and specify requirements in your prompt for best results
